# 在线聊天系统 - 团队分工方案（6 人）

---

## 分工原则

- 每人负责 **1 个独立模块**，涵盖该模块的前端 + 后端 + 数据库表设计
- 每个模块都有实质性的后端开发内容
- 模块间通过约定的接口协作，可并行开发
- 公共部分（项目脚手架、统一认证、WebSocket 基础设施）由负责人搭建后全员复用

---

## 模块划分总览

| 成员 | 模块 | 核心职责 | 工作量 |
|------|------|----------|--------|
| **A** | 用户与认证 | 注册、登录、个人中心、在线状态 | ★★★☆ |
| **B** | 私聊与实时通信 | 1v1 聊天、消息收发、已读状态、离线消息 | ★★★★ |
| **C** | 群聊 | 群 CRUD、群成员管理、群消息、群公告 | ★★★★ |
| **D** | 好友与分组 | 好友申请、分组管理、黑名单、好友搜索 | ★★★☆ |
| **E** | 消息中心与文件 | 聊天记录查询/搜索/导出、文件/图片/表情 | ★★★☆ |
| **F** | 语音与管理后台 | 语音消息、语音通话、管理员功能 | ★★★☆ |

---

## 各模块详细职责

### 模块 A — 用户与认证

**负责搭建项目公共基础（脚手架、数据库初始化、统一响应格式）**

#### 后端
- 用户注册接口（用户名唯一校验、密码加密）
- 用户登录接口（Token 签发与刷新）
- 用户登出接口（Token 失效）
- 获取/更新个人信息接口
- 修改密码接口（验证旧密码）
- 在线状态管理（上线、下线、状态切换）
- Token 认证拦截器（供其他模块复用）
- WebSocket 握手认证

#### 前端
- 登录页面
- 注册页面
- 个人信息编辑页（头像上传、昵称、签名）
- 修改密码页
- 在线状态切换 UI

#### 数据库
- `user` 表设计

#### 公共职责
- 搭建项目骨架（目录结构、统一异常处理、统一响应封装）
- 初始化数据库脚本
- 提供认证工具类供其他模块使用

---

### 模块 B — 私聊与实时通信

**负责搭建 WebSocket 通信基础（供群聊和语音复用）**

#### 后端
- WebSocket 连接管理（建立、断开、心跳检测）
- 私聊消息发送接口（实时推送 + 持久化）
- 消息已读 / 未读状态更新
- 离线消息存储与上线后拉取
- 在线用户列表维护
- 消息送达确认机制

#### 前端
- 主界面布局（左侧列表 + 右侧聊天窗口）
- 私聊聊天窗口（消息气泡、时间戳、已读标记）
- 消息输入框（文字发送）
- 实时消息接收与展示
- 历史消息滚动加载
- 未读消息角标

#### 数据库
- `private_message` 表设计

#### 公共职责
- 搭建 WebSocket 通信层（供群聊、语音模块复用）
- 消息收发协议定义（消息格式约定）

---

### 模块 C — 群聊

#### 后端
- 创建群聊接口
- 群信息查询 / 更新接口（群名、头像、公告）
- 邀请成员 / 踢出成员接口
- 成员角色管理（设为管理员、转让群主）
- 群消息发送接口（实时推送给所有群成员）
- 群消息免打扰设置
- 群成员列表查询
- 解散群聊接口

#### 前端
- 创建群聊弹窗 / 页面
- 群聊聊天窗口（与私聊复用基础 UI）
- 群成员管理面板（成员列表、角色标识、操作按钮）
- 群设置页（群名、公告、头像编辑）
- 群邀请好友弹窗
- 群列表展示（在主界面左侧）

#### 数据库
- `chat_group` 表
- `group_member` 表
- `group_message` 表

---

### 模块 D — 好友与分组

#### 后端
- 搜索用户接口（用户名 / 昵称模糊搜索）
- 发送好友申请接口（附带验证信息）
- 获取好友申请列表接口
- 接受 / 拒绝好友申请接口
- 删除好友接口
- 获取好友列表接口（按分组返回）
- 好友分组 CRUD 接口
- 移动好友到指定分组接口
- 重新发送验证信息接口
- 拉黑 / 取消拉黑接口

#### 前端
- 搜索用户页面 / 弹窗
- 好友申请列表页（待处理、已发送）
- 好友列表展示（分组折叠 / 展开）
- 分组管理（创建、重命名、删除分组）
- 右键菜单（移动分组、删除好友、发送消息、查看资料、拉黑）
- 好友资料查看弹窗

#### 数据库
- `friendship` 表
- `friend_group` 表

---

### 模块 E — 消息中心与文件

#### 后端
- 私聊聊天记录查询接口（分页、时间范围）
- 群聊聊天记录查询接口（分页、时间范围）
- 聊天记录关键词搜索接口
- 聊天记录导出接口（生成 TXT / CSV 文件供下载）
- 文件上传接口（图片、通用文件）
- 文件下载接口
- 表情列表接口（可选）

#### 前端
- 聊天记录查看页面（时间线展示）
- 搜索栏（关键词搜索，结果高亮跳转）
- 时间范围筛选器
- 导出按钮（选择格式 → 下载文件）
- 聊天中发送图片（上传 → 预览 → 发送）
- 聊天中发送文件（上传 → 进度条 → 发送）
- 表情选择器（可选）

#### 数据库
- `file_resource` 表（上传文件记录，可选）

---

### 模块 F — 语音与管理后台

#### 后端 — 语音
- 语音消息上传接口
- 语音文件存储与访问
- 语音通话信令服务（呼叫、接听、挂断、忙线）
- 语音通话状态管理

#### 后端 — 管理员
- 用户列表接口（分页、搜索）
- 禁用 / 启用用户接口
- 重置用户密码接口
- 群聊列表接口
- 解散群聊接口
- 系统公告 CRUD 接口
- 敏感词配置接口

#### 前端 — 语音
- 语音消息录制按钮（长按录音 → 发送）
- 语音消息播放组件（点击播放、进度条、时长显示）
- 语音通话 UI（呼叫中、通话中、挂断按钮）

#### 前端 — 管理员
- 管理后台入口与布局
- 用户管理页（列表、搜索、禁用/启用、重置密码）
- 群聊管理页（列表、解散）
- 系统公告管理页
- 敏感词管理页

#### 数据库
- `voice_message` 表
- `announcement` 表（系统公告，可选）
- `sensitive_word` 表（敏感词，可选）

## 协作约定

### 接口约定（全员对齐）
- 统一响应格式：`{ code, message, data }`
- 认证方式：请求头携带 Token
- WebSocket 消息格式：`{ type, from, to, content, timestamp }`
- 分页参数：`pageNum`、`pageSize`

### 开发顺序建议

```
第 1 周  A 搭建脚手架 + 用户认证  ←  全员等待公共基础
第 2 周  B 搭建 WebSocket 基础  +  D 好友模块  ←  并行
第 3 周  B 私聊功能完善  +  C 群聊开始  ←  并行
第 4 周  C 群聊功能完善  +  E 消息记录与文件  ←  并行
第 5 周  F 语音功能  +  E 导出功能  ←  并行
第 6 周  F 管理后台  +  全员联调测试  +  部署
```

### 联调节点

| 节点 | 时间 | 内容 |
|------|------|------|
| 联调 1 | 第 2 周末 | 用户登录 + WebSocket 连通 |
| 联调 2 | 第 3 周末 | 私聊 + 好友功能联调 |
| 联调 3 | 第 4 周末 | 群聊 + 记录导出联调 |
| 联调 4 | 第 5 周末 | 语音功能联调 |
| 总联调 | 第 6 周 | 全功能集成测试 + 部署上线 |

---

## 提交材料对照

根据课程要求，每人需确保有 **实质性后端开发内容**，以上分工中每个模块均包含：
- 独立的后端接口开发（不少于 3-5 个接口）
- 对应的数据库表设计
- 对应的前端页面开发
- 可独立演示的功能闭环

避免出现"只做前端"或"只做数据库设计"的情况。
